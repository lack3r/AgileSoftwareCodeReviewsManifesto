Agile Software Code Reviews Manifesto




We are uncovering better ways of reviewing
code by doing it and helping others do it.
Through this work we have come to value:

Quality focused development over following a checklist
 Simple code over generic solutions
 Early code reviews over last minute code clean up
 Code review over trusting comments
 Prefer immutability over changing state
 Reviewing in pairs over reviewing alone
 Document public API over private API
 Collective code ownership over individual code ownership
 Knowledge transfer over mute code reviews
 Working features over code coverage

That is, while there is value in the items on
the right, we value the items on the left more.
